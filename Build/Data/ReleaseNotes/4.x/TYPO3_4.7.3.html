Upgrading

The usual upgrading procedure applies. No database updates are necessary.

Changes

Here is a list of what was fixed since 4.7.2:

  • [RELEASE] Release of TYPO3 4.7.3
  • [BUGFIX] sectionIndex menu is not i18n ready (#36616)
  • [BUGFIX] Exception “Could not create directory” (#39583)
  • [BUGFIX] Incorrect search-results when searching for part of word (#38548)
  • [BUGFIX] Pass $fieldName when processing FlexForm DS in t3lib_transferData (#39527)
  • [BUGFIX] t3lib_db - expects parameter 1 to be resource, boolean given (#39509)
  • [BUGFIX] IRRE childs don't expand in Internet Explorer (#38849)
  • [BUGFIX] t3lib_db::exec_SELECTgetRows method annotation (#39417)
  • [BUGFIX] BE User Settings cannot be saved by clicking enter (#39203)
  • [BUGFIX] RTE: Installation of AllowClipboardHelper is not triggered (#39338)
  • [BUGFIX] Exclude E_STRICT from errors with PHP 5.4 (#38691)
  • [TASK] Always return a boolean in t3lib_div::validPathStr (#38604)
  • [BUGFIX] Invalid fallback for non-localized labels (#39220)
  • [BUGFIX] YouTube videos can not be played with Media CE (#37967)
  • [BUGFIX] stdWrap numRows fails due to wrong SELECT clause (#34152)
  • [TASK] Raise submodule pointer
  • [TASK] Change @deprecated annotation to the correct version (#39067)
  • [BUGFIX] QT movies prefixed abusively in Media CE (#39026)
  • [BUGFIX] Send sane HTTP response in showpic on error (#39052)
  • [BUGFIX] Remove bogus template in template analyzer (#38104)
  • [BUGFIX] accessibilityWrap ignores simple value (#38791)
  • [BUG] t3lib_div::getFilesInDir order differs from order in File list (#18771)
  • [BUGFIX] RTE: Importing google webfonts breaks style sheet parsing (#36316)
  • [BUGFIX] RTE spellcheck issue on Windows server (#36438)
  • [BUGFIX] RTE 4.7: Incorrect behaviours in IE9 native mode (#38657)
  • [TASK] Set TYPO3 version to 4.7.3-dev
